关于位操作符

移位操作符

移位操作符的运算对象是二进制的位，只能操作java的基础类型int，它的 二进制位数是32。
移位操作符有几种：

左移位操作符 <<

左移操作符将操作符左边的操作数向左移动操作符右侧指定的位数，在低位补0。

右移位操作符 >>

右移位操作符按照不同的场景有不同的处理方式，对于有符号数，右移操作符将操作符左侧的整型数向右移动操作符右侧的位数。
有符号数的右移操作符使用“符号扩展”：
1）若符号为正，则在高位插入0；
2）若符号为负，则在高位插入1；

无符号的右移操作符 >>>

无符号的右移操作符使用的是“零扩展”，无论政府，都在高位插入0；

如果对char、byte和short类型的数值类型进行移位操作，在移位之前，会被转换为int类型，其结果也是int类型。只有数值右端的
低5位才生效，这样可防止移位超过int类型所具有的位数。若对long类型数值进行处理，那么结果也是long，此时只会对数值右端
的低6位生效，以防止移位超过long型数值具有的位数。

操作符<<== 与 >>==

此时，操作之左边的值会移动由右边的值指定的位数，然后将结果赋给左边的变量。

如果对byte或者short的值进行移位操作，得到的结果可能不正确。因为它们先被转换成int类型，然后进行移位做操，然后被截断
赋值给运来的类型。

代码示例

public class Shift {
    public static void main(String[] args){
        int m = 1011;
        System.out.println("m->" + Integer.toBinaryString(m));
        System.out.println("m left shift 3 ->" + Integer.toBinaryString(m << 3));
        System.out.println("m right shift 3 ->" + Integer.toBinaryString(m >> 3));

        int n = -1011;
        System.out.println("n->" + Integer.toBinaryString(m));
        System.out.println("n left shift 3 ->" + Integer.toBinaryString(m << 3));
        System.out.println("n right shift 3 ->" + Integer.toBinaryString(m >> 3));

        int i = -1;
        System.out.println("i ->" + Integer.toBinaryString(i));
        i >>>= 10;
        System.out.println("i >>>= 10" + Integer.toBinaryString(i));

        long l = -1;
        System.out.println("l ->" + Long.toBinaryString(l));
        l >>>= 10;
        System.out.println("l >>>= 10 ->" + Long.toBinaryString(l));

        short s = -1;
        System.out.println("s ->" + Integer.toBinaryString(s));
        l >>>= 10;
        System.out.println("s >>>= 10 ->" + Integer.toBinaryString(s));

        byte b = -1;
        System.out.println("b ->" + Integer.toBinaryString(b));
        l >>>= 10;
        System.out.println("b >>>= 10 ->" + Integer.toBinaryString(b));

        b = -1;
        System.out.println("b >>> 10 ->" + Integer.toBinaryString(b >>> 10));

    }
}
